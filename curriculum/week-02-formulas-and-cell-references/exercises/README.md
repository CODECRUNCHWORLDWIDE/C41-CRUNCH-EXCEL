# Week 2 — Exercises

Three guided exercises, ~45–90 min each. **Type every formula yourself** — don't copy-paste values in from this document. Predicting a formula's result before you press Enter, and being wrong sometimes, is what actually builds the reference-tracking instinct this week is about.

1. **[Exercise 1 — Operator & precedence drills](exercise-01-operator-precedence-drills.md)** — predict-then-check arithmetic, order of operations, and the five aggregate functions.
2. **[Exercise 2 — Lock a rate with absolute refs](exercise-02-lock-the-tax-rate.md)** — build a priced order sheet and correctly lock a shared tax rate so it survives filling down.
3. **[Exercise 3 — Name ranges for readability](exercise-03-name-your-ranges.md)** — take a working-but-cryptic sheet and rewrite its formulas using named ranges.

## Before you start

- You've completed all three lectures.
- You have a blank workbook open (`crunch-week2.xlsx` or your Sheets equivalent from the week README's setup step).
- You know how to toggle formula view (`Ctrl+`` `` in both Excel and Sheets) — use it constantly while checking your work.

## Suggested workflow

- Open the exercise file beside your spreadsheet.
- For each task, **write down (or say out loud) what you expect the result to be before you type the formula**. If your prediction is wrong, stop and figure out why before moving on — that's the actual lesson, not the correct answer.
- Keep every exercise in its own sheet tab within the same workbook (`Ex1`, `Ex2`, `Ex3`) so you build one file you can reference all week.
- Use the fill handle and `Ctrl+D`/`Ctrl+R` deliberately — don't retype formulas cell by cell once you understand what should happen when you fill.

## Note on the two apps

Every task works identically in Excel and Google Sheets — the formula syntax, operators, and functions covered this week are shared between them. Where a keyboard shortcut differs (rare this week), the task calls it out with both spellings.
